# Isometric Workflow

## When To Use Isometric

Use isometric only if it supports the game idea.

Good reasons:

* business simulation layout
* office/shop/warehouse map
* grid-based decisions
* spatial planning
* resource flow visualization
* movement between stations or departments

Avoid isometric if the game is mostly:

* quiz/dialog
* dashboard analysis
* single-screen decision cards
* chart interpretation

## Workflow

1. Start with a high-quality source reference.
2. Generate a full-body anchor.
3. Convert to an isometric game-ready anchor.
4. Generate cardinal directions first: north, east, south, west.
5. Review identity and scale.
6. Generate diagonal directions second using the anchor plus cardinals.
7. Assemble directions in a clear order.
8. Normalize height and foot alignment.
9. Remove or key out background.
10. Test direction switching in a preview.

## Cardinal Directions First

Do not start with all eight directions if consistency is unstable.

Cardinals first give the model stronger structure and make it easier to inspect:

* north
* east
* south
* west

## Diagonals Second

Use the approved anchor and cardinal sheet to create:

* northeast
* southeast
* southwest
* northwest

Preserve:

* silhouette
* proportions
* asymmetrical details
* lighting
* camera angle
* foot position

## Normalization

Normalize all directions so the character does not shrink, grow, or bounce while turning.

Check:

* shared visible height
* shared bottom anchor
* consistent canvas size
* transparent background
* correct direction labels

## Safety Rule

Isometric reference projects are workflow references only. Do not copy their characters, setting, tactics style, or art identity.
