# Analyst Triage Prototype Task Skill

## When to use

Use this skill for any task that builds or modifies Analyst Triage prototype gameplay, company navigation, analysis tools, recommendations, or consequences.

## Core intent

Build a real game-first prototype, not a quiz, dashboard, or case-study UI.

## Required files to read

Read only what is needed. Usually:

- `docs/game-design/PROTOTYPE_V1_PLAYABLE_SPEC.md`
- `docs/game-design/FEEDBACK_ANIMATION_RULEBOOK.md`
- `docs/codex/NO_GO_AND_ACCEPTANCE_CRITERIA.md`
- relevant `src/` files

Do not read `references/` unless explicitly asked.

## Before coding

State:

- Goal
- Files to read
- Files to edit
- Constraints
- Done when

Keep this short.

## Design rules

- Player must do tangible actions.
- Every action must have feedback.
- Statistics must be revealed through interaction, not text explanation.
- Do not reveal the correct recommendation directly.
- Do not turn analysis into table reading.
- Do not add long dialogue.
- Do not expand scope beyond the requested task.

## Allowed prototype placeholders

- simple shapes for employees;
- cards for Sales/HR/recommendations;
- basic office rectangles;
- simple meters;
- basic animation/motion.

## Forbidden in prototype tasks

- final asset generation;
- large map;
- multiple missions unless requested;
- full dashboard;
- formulas as main UI;
- broad refactor;
- references scan.

## Completion report

Report:

- changed files;
- how the player action works;
- what feedback appears;
- what was not included;
- validation performed.
