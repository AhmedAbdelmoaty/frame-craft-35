# Project Walk Cycle Pipeline

Use this skill for project character walk animations. It narrows the general spritesheet workflow into the fastest safe path for this repo.

## Hard Rules

- Do one character, one animation, one direction per task.
- Do not generate all directions together.
- Do not generate idle and walk together.
- Do not run dev server or build for asset-only work.
- Do not use naive equal-grid crop from generated strips.
- Always recover frames/components, normalize cells, create GIF preview, and create frame review before indexing.
- Mark results as `candidate-review` until the user visually approves them.
- Stop after one failed generation and regenerate once with a narrower prompt. Do not keep spending credits.

## Required Inputs

Before generation, read only the minimum relevant files:

- `PROJECT_BRIEF.md`
- `assets/ASSET_INDEX.md`
- Current character anchor/turnaround from the asset index.
- For Male Analyst: `assets/generated/characters/player-turnarounds/v2-male/male-analyst-directional-turnaround-v2.png`.

Use the global sprite/animation skills when a broader workflow decision is needed.

## Direction Order

For a new character:

1. `walk-left`
2. `walk-right`
3. `walk-down`
4. `walk-up`
5. directional idle animations after walk candidates are visually approved

Do not parallelize directions for the same character until style, scale, and recovery settings are stable.

## Prompt Template

Use one prompt per direction. Keep it short and specific.

```text
Create a clean 8-frame WALK-[DIRECTION] animation strip for [CHARACTER].
Same identity in every frame: [age, face, hair/hijab, glasses, outfit, shoes, key accessories].
[Direction-specific camera rule].
White clean background, no text, no labels, no props, no motion blur.
8 separate full-body frames in one horizontal row, equal spacing, same scale, same camera, feet fully visible, hands fully visible.
Natural slow office walking, not running.
Real walk cycle poses with alternating legs: contact pose, passing pose, opposite contact, passing pose, then loop completion.
Arms swing naturally opposite the legs.
Body height stable with subtle bob only.
Keep identity, outfit colors, side-specific accessories, and proportions identical.
No jitter, no duplicate still frames, no warped limbs, no missing body parts, no cropped feet, no changed face.
```

### Male Analyst Accessory Rules

- Watch belongs to the character's left wrist.
- `walk-left`: visible near-side wrist may show the watch if it is anatomically the left wrist.
- `walk-right`: visible near-side right wrist must not show a watch; keep the left wrist hidden/far-side.
- `walk-down`: front view, character's left wrist appears on viewer's right side.
- `walk-up`: back view, verify left/right visually before accepting.

## Recovery Command

After saving the raw strip, run:

```powershell
python scripts\recover_spritesheet_strip.py `
  --src assets\generated\characters\player-spritesheets\male\v1\<animation>\male-analyst-<animation>-strip-v1.png `
  --out-dir assets\generated\characters\player-spritesheets\male\v1\<animation> `
  --prefix male-analyst-<animation>-v1 `
  --frames 8 `
  --duration-ms 140
```

Use the script output cell size for `assets/ASSET_INDEX.md`.

## Review Checklist

Reject before indexing if any are true:

- Still-image shake instead of real walking.
- Running rather than slow office walk.
- Cropped hands or feet after recovery.
- Face, hair, outfit, or scale changes strongly between frames.
- Wrong accessory side, especially Male Analyst watch.
- Motion blur, warped limbs, duplicated limbs, missing body parts.
- Less or more than 8 recovered components.

Candidate is acceptable when:

- GIF reads as a walk.
- Frame review shows complete bodies.
- Feet alternate clearly enough for the direction.
- Cell size is fixed and recorded.
- User can review before the next direction starts.

## Review File Minimum

Include:

- Status: `candidate-review`.
- Raw strip path.
- Recovered fixed-cell strip path.
- GIF preview path.
- Frame review path.
- Frame count and cell size.
- What works.
- Watch/failure notes.
- Whether transparency cleanup is still needed.

Keep it short. Do not write long process essays.
