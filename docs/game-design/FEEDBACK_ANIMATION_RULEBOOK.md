# Analyst Triage — Feedback & Animation Rulebook

## 1. Principle

Every player action must produce feedback. No silent interactions.

Feedback can be:

- visual;
- motion;
- sound cue;
- NPC micro-reaction;
- meter change;
- unlocked tool;
- team behavior;
- environmental pressure.

## 2. Feedback Table

| Player Action | Required Feedback | Meaning |
|---|---|---|
| Enter company | meeting alert, ambient movement | this is a live workplace |
| Speak to Sales | Sales pressure card appears, Team A badge pulses | business pressure favors surface number |
| Speak to HR | threshold tool unlocks, fairness meter appears | decision must be defensible |
| Enter Analyst Office | board activates, teams appear dimmed | analysis phase starts |
| Open Sales card | mean indicator revealed | surface metric becomes explicit |
| Scan team | units reveal gradually | player is uncovering data |
| Sort team | units move to performance axis | distribution becomes visible |
| Place threshold | line cuts across team, units above/below react | standard is now spatial |
| Mark outlier | selected units glow/outline | exceptional values isolated |
| Impact test | mean indicator pulses/shifts, body marker appears | metric sensitivity exposed |
| Drag reward card | target team responds in anticipation | recommendation has weight |
| Drag training card | target team shifts toward training mode | intervention assigned |
| Confirm decision | timer stops, world evaluates | commitment moment |
| Correct outcome | group celebration, trust/fairness rise | decision is defensible |
| Wrong outcome | objections, morale drop, trust/fairness fall | decision fails in world |

## 3. Animation Tone

Professional, mature, not childish.

Use:

- subtle pulses;
- smooth movement;
- clean transitions;
- readable contrast;
- office ambience.

Avoid:

- cartoon explosions;
- comedic failure effects;
- childish sounds;
- excessive particle effects;
- fantasy visual metaphors.

## 4. Data Unit Animation

Employee units must feel alive but not distracting.

States:

- hidden;
- revealed;
- sorted;
- above threshold;
- below threshold;
- marked as exceptional;
- rewarded;
- trained;
- objecting;
- improving.

## 5. Correct Outcome Animation

Correct decision should show:

- Team B group-wide positive reaction;
- Team A moves into structured training;
- below-threshold units improve slightly;
- HR closes objection panel;
- management trust rises;
- Sales pressure reduces.

## 6. Wrong Outcome Animation

Wrong decision should show:

- only Team A outlier units celebrate;
- Team A body stays below threshold and shows concern;
- Team B loses morale;
- HR objection panel opens;
- trust/fairness decline;
- training assigned to stable team has weak effect.

## 7. Audio Guidelines

Audio is optional in early V1 but recommended for feel.

Allowed:

- meeting alert tick;
- soft card pickup;
- scan pulse;
- sort whoosh;
- threshold click;
- approval chime;
- objection low tone.

Avoid loud, childish, or arcade-like sound effects.

## 8. Minimum V1 Animation Requirement

V1 does not need final art, but it must include movement feedback for:

- scan reveal;
- sort arrangement;
- threshold distinction;
- recommendation assignment;
- correct/incorrect outcome.

If these are missing, V1 is not a game prototype yet.
